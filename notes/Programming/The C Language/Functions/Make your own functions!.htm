Why do we want to make functions?

  • Don't have to copy and paste repeated code!
    Reuse parts of our code
  • Make our main function smaller and simpler

Make our programs nicer to read

 

What do functions do?

  • Separates (aka encapsulates) a piece of code serving a single purpose
  • Tests and verifies a piece of code
  • Reuse a piece of code
  • Shortens our programs, making it easier to modify and debug

 

Okay cool, where do I start???

The structure of a function

[return type] [function name] ([input type] [input name]) {

return [return value];

}

Example

Below is a code which accepts an integer parameter called `number`, and returns a value that is 1 larger than it.

 

/*

   * return type: int

 * function name: addOne

 * variables:

 *   - `number` with type int

*/

 

int addOne(int number) {

int newNumber = number + 1;

return newNumber;

}

 

> addOne(5) == 6

What if I don't need any parameters in my function???

The  paramater basically means that nothing is needed.

void doStuff(void) {

// stuff is done

}

 

Function Prototypes

These tell the compiler that the function exists, and the structure it has. It includes key information about the function

ie

int addNumbers(int num1, int num2);

 

A function can have zero or more parameters.

A function can only return zero or one value.

 

 

A function stores a local copy of parameters passed into it

The original values of the variables remain unaltered

Parameters received by the function, and local variables created by the function are all discarded when the function returns
